# 튜터보드(TutorBoard) 서비스 기획서

## 1. 서비스 개요

### 서비스명

튜터보드(TutorBoard)

### 한 줄 소개

과외 선생님의 수업 메모를 기반으로 학생별 진도 리포트, 학부모 안내 메시지, 결제 리마인드, 발송 대기열을 자동 생성하는 AI 과외 운영 보조 서비스입니다.

### 핵심 아이디어

1인 과외 선생님은 수업 준비, 수업 기록, 학부모 소통, 결제 안내, 일정 조정까지 혼자 처리해야 합니다. 튜터보드는 선생님이 짧은 수업 메모와 결제/일정 정보를 입력하면 Solar Pro 기반 AI Agent가 학부모에게 전달하기 좋은 문장과 구조화된 운영 데이터를 자동 생성합니다.

최종 결과물은 선생님용 대시보드와 학생/학부모용 대시보드에서 확인할 수 있으며, 리포트와 결제 안내 메시지는 이메일 또는 카카오 공유 형태로 발송할 수 있습니다.

## 2. 문제 정의

### 대상 사용자

- 1인 과외 선생님
- 소규모 개인 교습 운영자
- 학생과 학부모

### 현재 문제

과외 선생님은 수업 이후 학부모에게 진도, 숙제, 취약 개념, 다음 수업 계획을 전달해야 하지만 매번 정리하는 데 시간이 많이 듭니다. 결제 안내 역시 학부모에게 부담스럽게 느껴질 수 있어 메시지 톤을 조절하기 어렵습니다. 일정 변경 요청도 카카오톡, 문자, 전화 등 여러 채널에 흩어져 관리되기 쉽습니다.

기존 학원 관리 SaaS는 출결, 수납, 공지 중심으로 설계되어 있어 1인 과외 선생님의 실제 운영 흐름과는 맞지 않는 경우가 많습니다. 과외 선생님에게 필요한 것은 거대한 관리 시스템보다, 수업 맥락을 이해하고 학부모와의 신뢰를 유지해주는 가벼운 AI 보조 도구입니다.

## 3. 서비스 목표

튜터보드의 목표는 과외 선생님의 반복 운영 업무를 줄이고, 학부모와의 소통 품질을 높이는 것입니다.

- 수업 메모를 학부모용 리포트로 자동 변환
- 학생별 진도, 취약 개념, 숙제 상태, 다음 계획을 구조화
- 결제 안내를 독촉이 아닌 수업 준비 안내 문맥으로 생성
- 학생/학부모가 확인할 수 있는 대시보드 제공
- 일정 변경 요청과 가능 시간 확인 흐름 제공
- 모든 AI Agent 결과를 JSON으로 고정하여 UI와 DB에 쉽게 연결

## 4. 핵심 기능

### 4.1 선생님용 기능

- 학생 목록 관리
- 학생별 수업 기록 입력
- AI 수업 리포트 생성
- 결제 상태 확인
- 결제 안내 메시지 생성
- 가능 일정 등록
- 일정 변경 요청 확인 및 승인
- 생성된 메시지 발송 대기열 확인

### 4.2 학생/학부모용 기능

- 학생별 대시보드 확인
- 최근 수업 리포트 확인
- 현재 진도와 취약 개념 확인
- 숙제 상태와 다음 수업 계획 확인
- 결제 안내 확인
- 일정 변경 요청

### 4.3 AI Agent 기능

튜터보드는 4개의 Agent로 구성됩니다.

1. Tutor Profile Agent
2. Lesson Report Agent
3. Payment Reminder Agent
4. Parent Communication Agent / Orchestrator

각 Agent는 반드시 JSON만 출력하며, 최종적으로 `message_queue` 형태로 UI와 DB에 전달됩니다.

## 5. Agent 설계

### 5.1 Tutor Profile Agent

과외 선생님의 운영 스타일을 구조화하는 Agent입니다.

입력 예시:

```json
{
  "subject": "고등 수학",
  "teaching_style": "꼼꼼하게 개념을 잡아주는 스타일",
  "payment_policy": "월 4회 선결제",
  "parent_tone": "정중하지만 부담스럽지 않게"
}
```

출력 예시:

```json
{
  "agent_profile": {
    "subject": "고등 수학",
    "tone": "정중하고 부담스럽지 않게",
    "communication_rules": [
      "학생을 부정적으로 단정하지 않는다",
      "문제점과 개선 계획을 함께 제시한다",
      "결제 안내는 수업 준비와 연결해 설명한다"
    ],
    "report_format": [
      "오늘의 진도",
      "취약 개념",
      "숙제 상태",
      "다음 수업 계획"
    ]
  }
}
```

역할:

- 선생님별 말투와 운영 기준 생성
- 이후 리포트/결제 안내 메시지의 기준값 제공
- 학부모 소통의 일관성 유지

### 5.2 Lesson Report Agent

선생님의 수업 메모를 학부모용 리포트로 변환하는 Agent입니다.

입력 예시:

```json
{
  "student": {
    "name": "김서윤",
    "grade": "고1",
    "subject": "수학"
  },
  "lesson_memo": "오늘 지수함수 그래프 이동 수업. 로그 개념 헷갈려함. 숙제 15문제 중 9문제 완료. 계산 실수 많음.",
  "agent_profile": {}
}
```

출력 예시:

```json
{
  "progress": "지수함수 그래프 이동",
  "weakness": "로그 개념 연결, 계산 실수",
  "homework_status": "15문제 중 9문제 완료",
  "next_plan": "로그 기본 개념 복습 후 지수·로그 연결 문제 풀이",
  "parent_report": "오늘은 지수함수의 그래프 이동을 중심으로 수업했습니다..."
}
```

역할:

- 수업 메모에서 핵심 진도 추출
- 취약 개념과 숙제 상태 정리
- 다음 수업 계획 생성
- 학부모에게 보낼 수 있는 친화적 문체로 변환

### 5.3 Payment Reminder Agent

결제 상태와 다음 수업 일정을 보고 결제 안내 메시지를 생성하는 Agent입니다.

입력 예시:

```json
{
  "student_name": "김서윤",
  "parent_name": "김서윤 학부모님",
  "payment_status": "unpaid",
  "payment_due_date": "2026-05-24",
  "amount": 320000,
  "class_count": 4,
  "next_class": "2026-05-24 19:00"
}
```

출력 예시:

```json
{
  "should_send": true,
  "message_type": "payment_reminder",
  "urgency": "normal",
  "message_body": "안녕하세요. 내일 수업이 예정되어 있어 수업 준비 차 안내드립니다..."
}
```

역할:

- 미결제 여부 확인
- 결제 안내 발송 필요 여부 판단
- 결제 메시지를 수업 준비 안내 문맥으로 완화
- 학부모가 부담을 덜 느끼는 문장 생성

### 5.4 Parent Communication Agent / Orchestrator

앞선 Agent 결과를 종합하여 최종 발송 대기열을 만드는 Agent입니다.

출력 예시:

```json
{
  "message_queue": [
    {
      "student_name": "김서윤",
      "message_type": "lesson_report",
      "channel": "email_mock",
      "status": "pending",
      "message_body": "오늘은 지수함수의 그래프 이동을..."
    },
    {
      "student_name": "김서윤",
      "message_type": "payment_reminder",
      "channel": "email_mock",
      "status": "pending",
      "message_body": "안녕하세요. 내일 수업이..."
    }
  ]
}
```

역할:

- 수업 리포트와 결제 안내를 하나의 메시지 큐로 통합
- UI와 DB에 저장 가능한 최종 JSON 생성
- 발송 채널과 상태값 관리

## 6. 서비스 차별점

### 기존 학원 관리 SaaS와의 차이

기존 학원 관리 SaaS는 출결, 수납, 공지, 상담 기록 등 학원 운영 관리 기능에 초점이 맞춰져 있습니다. 반면 튜터보드는 과외 선생님 한 명의 실제 업무 흐름에 맞춰 설계됩니다.

### AI 자동 문구 생성 서비스와의 차이

단순히 안내 문구를 생성하는 것이 아니라, 선생님의 운영 스타일, 학생의 수업 상태, 결제 상황을 함께 반영합니다. 즉, 문장 자동 생성이 아니라 과외 운영 맥락을 이해하는 Agent Builder입니다.

### 핵심 포지셔닝

튜터보드는 "1인 교육자를 위한 학부모 신뢰 유지 Agent"입니다. 관리 효율뿐 아니라 상담 전환, 학부모 불안 완화, 결제 안내의 부담 완화까지 함께 다룹니다.

### 7.4 화면 구성

MVP 화면은 선생님이 수업 메모를 입력하고, AI가 학부모용 리포트와 결제 안내를 생성한 뒤, 학부모 화면에서 결과를 확인하는 흐름을 중심으로 설계합니다.

#### 7.4.1 로그인 / 데모 계정 선택 화면

사용자:

- 과외 선생님
- 학생/학부모

화면 목적:

- 실제 회원가입 구현 전, 시연용 계정으로 역할별 화면에 진입합니다.

주요 UI:

- 서비스명과 간단한 설명
- `선생님으로 시작하기` 버튼
- `학부모로 보기` 버튼
- 데모 학생 선택 드롭다운

연결 데이터:

- `tutors`
- `students`

#### 7.4.2 선생님 대시보드

#### 7.4.9 일정 변경 요청 화면

사용자:

- 학생/학부모
- 과외 선생님

화면 목적:

- 학부모가 가능한 시간 중 원하는 보강/변경 시간을 요청하고, 선생님이 승인합니다.

주요 UI:

- 선생님 가능 시간 목록
- 요청 사유 입력
- `일정 변경 요청` 버튼
- 선생님용 승인/거절 버튼

연결 데이터:

- `schedules`
- `students`
- `tutors`

MVP에서는 일정 변경 요청 화면은 선택 구현으로 두고, 시간이 부족하면 학부모 대시보드에 버튼만 노출합니다.

## 9. 데모 시나리오

### 시연 데이터

- 선생님: 고등 수학 과외 선생님
- 학생: 김서윤, 고1, 수학
- 수업 메모: 오늘 지수함수 그래프 이동 수업. 로그 개념 헷갈려함. 숙제 15문제 중 9문제 완료. 계산 실수 많음.
- 결제 상태: 미결제
- 결제 예정일: 2026-05-24
- 다음 수업: 2026-05-24 19:00

### 시연 흐름

1. 선생님이 로그인한다.
2. 학생 목록에서 김서윤 학생을 선택한다.
3. 수업 메모를 입력한다.
4. AI 리포트 생성을 누른다.
5. 진도, 취약 개념, 숙제 상태, 다음 계획, 학부모용 메시지가 생성된다.
6. 결제 상태가 미결제이면 결제 안내 메시지가 함께 생성된다.
7. 최종 메시지 큐가 생성된다.
8. 학부모 화면에서 리포트와 결제 안내를 확인한다.

## 13. 리스크와 대응 방안

### 리스크 1: AI 출력 형식이 흔들릴 수 있음

대응:

- 모든 Agent에 JSON-only system prompt 적용
- output schema 고정
- mock 로직으로 기본 시연 가능하게 구성
- 테스트 케이스 3개 이상 준비

### 리스크 2: 범위가 넓어질 수 있음

대응:

- MVP는 수업 리포트와 결제 안내에 집중
- 녹음 요약, 실제 결제 연동, 실제 알림톡은 보류
- 일정 기능은 시간이 남을 때 선택 구현

### 리스크 3: 실제 발송 연동이 지연될 수 있음

대응:

- 이메일 mock 또는 카카오링크 mock으로 시연
- `message_queue`에 발송 상태를 저장하여 추후 실제 연동 가능하게 설계

## 14. 성공 기준

MVP 성공 기준은 다음과 같습니다.

- 선생님 정보 입력 시 `agent_profile`이 생성된다.
- 수업 메모 입력 시 `lesson_report`가 생성된다.
- 미결제 상태 입력 시 `payment_reminder`가 생성된다.
- 최종 결과가 `message_queue` JSON으로 통합된다.
- 생성된 결과가 선생님 화면과 학부모 화면에서 확인된다.
- 데모 데이터 기준으로 전체 플로우가 끊기지 않고 시연된다.

## 15. 결론

튜터보드는 1인 과외 선생님의 반복적인 학부모 소통과 운영 업무를 줄이기 위한 AI Agent 서비스입니다. 핵심은 수업 메모를 단순 요약하는 데 그치지 않고, 선생님의 운영 스타일과 학생의 학습 상태, 결제 상황을 함께 반영해 실제로 발송 가능한 메시지와 대시보드 데이터를 생성하는 것입니다.

이를 통해 과외 선생님은 수업 품질에 더 집중할 수 있고, 학부모는 자녀의 학습 상황을 더 명확하게 이해할 수 있습니다. MVP는 Solar Pro 기반 Agent 파이프라인과 Supabase 저장, 선생님/학부모 대시보드를 중심으로 구현하며, 이후 실제 발송 채널과 일정 자동화 기능으로 확장할 수 있습니다.
